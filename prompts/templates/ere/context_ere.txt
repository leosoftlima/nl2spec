Context:
The IR represents an Extended Regular Expression (ERE) runtime monitor.

Event structure:
ERE IR structure:
- "formalism" must be "ere".
- "ir.ere" must store the expression in "expression".
- "ir.violation" must include "tag", "body.statements", and "body.has_reset".
- Each event in "ir.events.body.methods" must preserve:
  "action", "name", "timing", "parameters", "procediments", "function", and "operation".
- Include "returning" only when present.
- Use "negated": true only when the pointcut explicitly contains negation such as !cflow(...), !target(...), or !call(...).

Pointcut operator rule:
- "function" stores the top-level pointcut atoms in order.
- "operation" stores the logical operators between consecutive atoms.
- If "function" has N items, "operation" must have N-1 items.
- Operators inside condition(...) remain only inside the condition string.

Constraints:
- Do not invent events, methods, operators, or violation statements.
- Preserve names, parameters, timing, pointcut structure, operators, and negations exactly.
- Use only the declared event names in "ir.ere.expression".
- Follow the structure shown in the examples.
