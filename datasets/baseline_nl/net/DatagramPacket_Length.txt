This specification is written in the EVENT formalism.

This specification is called DatagramPacket_Length.


The following events are monitored in this specification:

construct_len is an event observed before the call. It takes byte[] buffer and int length as parameters. It matches: call(DatagramPacket.new(byte[], int)) || call(DatagramPacket.new(byte[], int, InetAddress, int)) || call(DatagramPacket.new(byte[], int, SocketAddress)) && args(buffer, length, ..).

construct_offlen is an event observed before the call. It takes byte[] buffer, int offset, and int length as parameters. It matches: call(DatagramPacket.new(byte[], int, int)) || call(DatagramPacket.new(byte[], int, int, InetAddress, int)) || call(DatagramPacket.new(byte[], int, int, SocketAddress)) && args(buffer, offset, length, ..).

A violation occurs when a monitored event is triggered under the pointcut conditions described above and its associated violation handler is activated.

The violation message that must be used in the JSON output is:

""The offset argument and/or the length argument are out of range; " + offset + " + " + length + " > " + buffer.length"

The generated JSON must include the field:

"domain": "net"
