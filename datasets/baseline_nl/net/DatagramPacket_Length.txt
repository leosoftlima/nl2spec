This specification is written in the EVENT formalism.

This specification is called DatagramPacket_Length.


The following events are monitored in this specification:

construct_len is observed before the call as a event. It takes byte[] buffer and int length. It matches: call(DatagramPacket.new(byte[], int)) || call(DatagramPacket.new(byte[], int, InetAddress, int)) || call(DatagramPacket.new(byte[], int, SocketAddress)) && args(buffer, length, ..).

construct_offlen is observed before the call as a event. It takes byte[] buffer, int offset, and int length. It matches: call(DatagramPacket.new(byte[], int, int)) || call(DatagramPacket.new(byte[], int, int, InetAddress, int)) || call(DatagramPacket.new(byte[], int, int, SocketAddress)) && args(buffer, offset, length, ..).

A violation occurs when a monitored event is triggered under the pointcut conditions described above and its associated violation handler is activated.

The violation message that must be used in the JSON output is:

"msg"

The generated JSON must include the field:

"domain": "net"

Return ONLY the IR JSON.
Do not include explanations, markdown, comments, or extra text.
