This specification is written in the EVENT formalism.

This specification is called InetSocketAddress_Port.


The following events are monitored in this specification:

construct is an event observed before the call. It takes int port as parameter. It matches: call(InetSocketAddress.new(int)) || call(InetSocketAddress.new(InetAddress, int)) || call(InetSocketAddress.new(String, int)) || call(* InetSocketAddress.createUnresolved(String, int)) && args(.., port).

A violation occurs when a monitored event is triggered under the pointcut conditions described above and its associated violation handler is activated.

The violation message that must be used in the JSON output is:

"The specified port " + port + " is out of range; [0 ~ 65535]"

The generated JSON must include the field:

"domain": "net"
