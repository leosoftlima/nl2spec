This specification is written in the EVENT formalism.

This specification is called Object_NoClone.


The following events are monitored in this specification:

clone is an event observed before the call. It takes Object o as parameter. It matches: call(* Object.clone()) && within(org.apache.xerces.dom.NodeImpl) && within(org.apache.batik.bridge.UpdateManager) && within(org.codehaus.janino.ScriptEvaluator) && within(org.apache.batik.ext.awt.image.codec.png.PNGEncodeParam) && within(org.apache.batik.ext.awt.image.spi.MagicNumberRegistryEntry) && within(org.apache.xerces.util.XMLCatalogResolver) && target(o) && condition(o.getClass().getCanonicalName().equals("java.lang.Object")).

A violation occurs when a monitored event is triggered under the pointcut conditions described above and its associated violation handler is activated.

The violation message that must be used in the JSON output is:

"Using the clone method on an Object instance will result in throwing an exception."

The generated JSON must include the field:

"domain": "lang"

Return ONLY the IR JSON.
Do not include explanations, markdown, comments, or extra text.
