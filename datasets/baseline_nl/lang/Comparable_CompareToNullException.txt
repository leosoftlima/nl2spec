This specification is written in the EVENT formalism.

This specification is called Comparable_CompareToNullException.


The following events are monitored in this specification:

badexception is observed after the call as a event. It takes Object o) throwing(Exception e. It matches: call(* Comparable+.compareTo(..)) && args(o) && if(o == null) && condition(!(e instanceof NullPointerException)).

badcompare is observed after the call as a event. It takes Object o. It captures the returned value as int i. It matches: call(* Comparable+.compareTo(..)) && args(o) && if(o == null).

A violation occurs when a monitored event is triggered under the pointcut conditions described above and its associated violation handler is activated.

The violation message that must be used in the JSON output is:

"__DEFAULT_MESSAGE"
