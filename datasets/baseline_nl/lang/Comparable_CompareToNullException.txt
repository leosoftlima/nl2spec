This specification is written in the EVENT formalism.

This specification is called Comparable_CompareToNullException.


The following events are monitored in this specification:

badexception is an event observed after the call. It takes Object o) throwing(Exception e as parameter. It matches: call(* Comparable+.compareTo(..)) && args(o) && if(o == null) && condition(!(e instanceof NullPointerException)).

badcompare is an event observed after the call. It takes Object o as parameter. It captures the returned value as int i. It matches: call(* Comparable+.compareTo(..)) && args(o) && if(o == null).

A violation occurs when a monitored event is triggered under the pointcut conditions described above and its associated violation handler is activated.

The violation message that must be used in the JSON output is:

"NullPointerException should be thrown when an object is compared to null"

The generated JSON must include the field:

"domain": "lang"

Return ONLY the IR JSON.
Do not include explanations, markdown, comments, or extra text.
