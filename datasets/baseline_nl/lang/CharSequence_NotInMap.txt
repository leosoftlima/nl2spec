This specification is written in the EVENT formalism.

This specification is called CharSequence_NotInMap.
It should use these parameters: [Map map].

The following events are monitored in this specification:

map_put is an event observed before the call. It takes Map map as parameter. It matches: call(* Map+.put(..)) && target(map) && args(CharSequence, ..) && args(String, ..) && args(CharBuffer, ..) && condition(!flag).

map_putall is an event observed before the call. It takes Map map and Map m as parameters. It matches: call(* Map+.putAll(Map)) && args(m) && target(map) && condition(!flag).

A violation occurs when a monitored event is triggered under the pointcut conditions described above and its associated violation handler is activated.

The violation message that must be used in the JSON output is:

"It might not be safe to add a CharSequence instance into a map as a key"

The generated JSON must include the field:

"domain": "lang"

Return ONLY the IR JSON.
Do not include explanations, markdown, comments, or extra text.
