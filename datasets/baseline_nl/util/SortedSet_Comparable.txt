This specification is written in the EVENT formalism.

This specification is called SortedSet_Comparable.
It should use these parameters: [SortedSet s].

The following events are monitored in this specification:

add is an event observed before the call. It takes SortedSet s and Object e as parameters. It matches: call(* Collection+.add*(..)) || call(* Queue+.offer*(..)) && target(s) && args(e).

addall is an event observed before the call. It takes SortedSet s and Collection c as parameters. It matches: call(* Collection+.addAll(Collection)) && target(s) && args(c).

A violation occurs when a monitored event is triggered under the pointcut conditions described above and its associated violation handler is activated.

The violation message that must be used in the JSON output is:

"A non-comparable object is being inserted into a SortedSet object."

The generated JSON must include the field:

"domain": "util"

Return ONLY the IR JSON.
Do not include explanations, markdown, comments, or extra text.
