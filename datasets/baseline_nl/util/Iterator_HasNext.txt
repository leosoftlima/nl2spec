This specification is called Iterator_HasNext.

It should use these parameters:
- Iterator i

Keep this name and parameter list exactly in the generated JSON.

The following events are monitored:

hasnexttrue is an event observed after the call. It takes Iterator i as parameter. It captures the returned value as boolean b. It matches: call(* Iterator+.hasNext()) && target(i) && condition(b)

hasnextfalse is an event observed after the call. It takes Iterator i as parameter. It captures the returned value as boolean b. It matches: call(* Iterator+.hasNext()) && target(i) && condition(!b)

next is an event observed before the call. It takes Iterator i as parameter. It matches: call(* Iterator+.next()) && target(i)

The temporal property that must hold during execution is:

ltl: [](next => (*) hasnexttrue)

In plain language, this means:

it must always hold that whenever next occurs, then hasnexttrue must have occurred in the previous step

If this temporal property is violated during execution, the JSON output must use the following violation behavior:

When the property is violation, the monitor logs __DEFAULT_MESSAGE and logs "Iterator.hasNext() was not called before calling next()."
