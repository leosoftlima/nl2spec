This specification is written in the EVENT formalism.

This specification is called TreeMap_Comparable.


The following events are monitored in this specification:

create is an event observed before the call. It takes Map src as parameter. It matches: call(TreeMap.new(Map)) && args(src).

put is an event observed before the call. It takes Object key as parameter. It matches: call(* Map+.put(Object, Object)) && args(key, ..) && target(TreeMap).

putall is an event observed before the call. It takes Map src as parameter. It matches: call(* Map+.putAll(Map)) && args(src) && target(TreeMap).

A violation occurs when a monitored event is triggered under the pointcut conditions described above and its associated violation handler is activated.

The violation message that must be used in the JSON output is:

"A non-comparable object is being inserted into a TreeMap object."

The generated JSON must include the field:

"domain": "util"

Return ONLY the IR JSON.
Do not include explanations, markdown, comments, or extra text.
