This specification is written in the EVENT formalism.

This specification is called InvalidPropertiesFormatException_NonSerializable.


The following events are monitored in this specification:

serialize is an event observed before the call. It takes ObjectOutputStream out and InvalidPropertiesFormatException obj as parameters. It matches: call(* ObjectOutputStream+.writeObject(..)) && target(out) && args(obj).

deserialize is an event observed after the call. It takes ObjectInputStream in as parameter. It captures the returned value as InvalidPropertiesFormatException obj. It matches: call(* ObjectInputStream+.readObject(..)) && target(in).

A violation occurs when a monitored event is triggered under the pointcut conditions described above and its associated violation handler is activated.

The violation message that must be used in the JSON output is:

"InvalidPropertiesFormatException is not intended to be Serialized."

The generated JSON must include the field:

"domain": "util"

Return ONLY the IR JSON.
Do not include explanations, markdown, comments, or extra text.
