The following events are involved in this specification:

1) Event name: create
   Kind: creation
   Timing: after
   Parameters: none
   Returning: ListIterator i
   Pointcut: call(ListIterator Iterable+.listIterator())

2) Event name: remove
   Kind: event
   Timing: before
   Parameters: ListIterator i
   Pointcut: call(void Iterator+.remove()) && target(i)

3) Event name: add
   Kind: event
   Timing: before
   Parameters: ListIterator i
   Pointcut: call(void ListIterator+.add(..)) && target(i)

4) Event name: next
   Kind: event
   Timing: before
   Parameters: ListIterator i
   Pointcut: call(* Iterator+.next()) && target(i)

5) Event name: previous
   Kind: event
   Timing: before
   Parameters: ListIterator i
   Pointcut: call(* ListIterator+.previous()) && target(i)

6) Event name: set
   Kind: event
   Timing: before
   Parameters: ListIterator i
   Pointcut: call(* ListIterator+.set(..)) && target(i)

The required execution pattern is defined by the following regular expression:

create add* ((next | previous)+ set* (remove | add+ | epsilon))*

Any execution trace that does not conform to this pattern constitutes a violation.

The violation message that must be used in the JSON output is:

"ListIterator.set() can be made only if neither remove() nor add() have been called after the last call to next() or previous()."
