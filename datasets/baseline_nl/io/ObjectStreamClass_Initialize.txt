This specification is called ObjectStreamClass_Initialize.

It should use these parameters:
- ObjectStreamClass c

Keep this name and parameter list exactly in the generated JSON.

The following events are monitored:

create is an event observed after the call. It does not take explicit event parameters. It captures the returned value as ObjectStreamClass c. It matches: call(ObjectStreamClass+.new())

init is an event observed before the call. It takes ObjectStreamClass c as parameter. It matches: call(* ObjectStreamClass+.initProxy(..)) && call(* ObjectStreamClass+.initNonProxy(..)) && call(* ObjectStreamClass+.readNonProxy(..)) && target(c)

endProg is an event observed before the call. It does not take explicit event parameters. It matches: endProgram()

The temporal property that must hold during execution is:

ltl: [](create => o init)

In plain language, this means:

it must always hold that whenever create occurs, then init occurs in the next step

If this temporal property is violated during execution, the JSON output must use the following violation behavior:

When the property is violation, the monitor logs __DEFAULT_MESSAGE and logs "An ObjectStreamClass object was instantiated, but was not initialized using initProxy(), initNonProxy() or readNonProxy()"

The generated JSON must include the field:

"domain": "io"

Follow the same JSON structure used in the examples.
Reuse the exact event names consistently.
Do not invent new events, operators, or fields.

Return ONLY the IR JSON.
