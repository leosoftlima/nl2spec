This specification is called ObjectInput_Close.

It should use these parameters:
- ObjectInput i

Keep this name and parameter list exactly in the generated JSON.

The following events are monitored:

create is an event observed after the call. It does not take explicit event parameters. It captures the returned value as ObjectInput i. It matches: call(ObjectInput+.new(..))

close is an event observed before the call. It takes ObjectInput i as parameter. It matches: call(* ObjectInput+.close(..)) && target(i)

endProg is an event observed before the call. It does not take explicit event parameters. It matches: endProgram()

The temporal property that must hold during execution is:

ltl: [](create => o close)

In plain language, this means:

it must always hold that whenever create occurs, then close occurs in the next step

If this temporal property is violated during execution, the JSON output must use the following violation behavior:

When the property is violation, the monitor logs __DEFAULT_MESSAGE and logs "close() was not called."

The generated JSON must include the field:

"domain": "io"

Follow the same JSON structure used in the examples.
Reuse the exact event names consistently.
Do not invent new events, operators, or fields.

Return ONLY the IR JSON.
